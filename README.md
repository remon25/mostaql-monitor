# Mostaql Monitor

أداة تراقب مشاريع "برمجة، تطوير المواقع والتطبيقات" الجديدة على مستقل، وتفلترها تلقائيًا حسب شرطين:

1. **الميزانية**: 250$ فأعلى (قابل للتعديل من `.env`)
2. **العميل**: معدل توظيف 30% فأعلى، أو عميل جديد (لم يُحسب بعد) — قابل للتعديل أيضًا
3. **عمر المشروع**: 15 دقيقة كحد أقصى منذ نشره — قابل للتعديل (`MAX_PROJECT_AGE_MINUTES`). بما أن قائمة المشاريع مرتبة من الأحدث للأقدم، بمجرد وصول الفحص لأول مشروع أقدم من هذا الحد، **يتوقف الفحص فورًا** (لا داعي لفحص باقي المشاريع الأقدم ولا الصفحات التالية).

عند مطابقة مشروع للشرطين، تحفظ الأداة تفاصيله + **مسودة عرض جاهزة** (باستخدام مهاراتك التي زودتني بها) في ملف `data/matches.json`، **دون إرسال أي شيء تلقائيًا** — المراجعة والإرسال يدويًا من طرفك.

## التركيب

```bash
npm install
cp .env.example .env   # ثم عدّل القيم إن أردت
npm start
```

الإعداد الافتراضي الآن: `RUN_MODE=cron` و`CRON_SCHEDULE=*/5 * * * *`، أي أن الأداة تفحص القائمة **كل 5 دقائق تلقائيًا** ولا تتوقف (اتركها تعمل في نافذة Terminal مفتوحة، أو استخدم أداة مثل `pm2` لتشغيلها في الخلفية بشكل دائم).

## إعداد إشعارات تيليجرام

1. افتح تطبيق تيليجرام وابحث عن `@BotFather`
2. أرسل له `/newbot` واتبع التعليمات (اختر اسمًا واسم مستخدم لبوتك)
3. سيعطيك BotFather **توكن** شكله مثل: `123456789:ABCdefGhIJKlmNoPQRstuVWXyz`
4. ضع هذا التوكن في `.env` بجانب `TELEGRAM_BOT_TOKEN=`
5. الآن تحتاج لمعرفة **Chat ID** الخاص بك:
   - ابدأ محادثة مع البوت الذي أنشأته (اضغط Start أو أرسل أي رسالة له)
   - افتح هذا الرابط في المتصفح (بعد استبدال التوكن):
     ```
     https://api.telegram.org/bot<التوكن>/getUpdates
     ```
   - ابحث في الرد عن `"chat":{"id":123456789` — هذا الرقم هو الـ Chat ID
   - ضعه في `.env` بجانب `TELEGRAM_CHAT_ID=`
6. شغّل `npm start` — أي مشروع مطابق للشروط سيصلك كإشعار مباشر على تيليجرام يحتوي: العنوان، الميزانية، اسم العميل، معدل التوظيف، وعدد العروض، مع رابط المشروع.

> ملاحظة: إذا لم يتم ضبط `TELEGRAM_BOT_TOKEN` أو `TELEGRAM_CHAT_ID`، ستستمر الأداة بالعمل وحفظ النتائج في `data/matches.json` كالمعتاد، لكنها ستطبع تحذيرًا في الـ Console بدل إرسال الإشعار.

## طريقة العمل

1. `src/index.js` يجلب قائمة المشاريع من `LISTING_URL`.
2. لكل مشروع جديد (لم تتم معالجته من قبل — يُحفظ في `data/seen.json`)، يفتح صفحة المشروع الفردية ويستخرج:
   - الميزانية
   - معدل توظيف صاحب المشروع
   - اسم العميل
3. يفلتر حسب الشروط، ويحفظ المشاريع المطابقة في `data/matches.json` مع نص عرض جاهز.
4. يمكن تشغيله مرة واحدة (`RUN_MODE=once`) أو بشكل دوري كل عدة دقائق (`RUN_MODE=cron`).

## ⚠️ ملاحظة مهمة جدًا حول الاستخراج

لم أتمكن من فتح صفحة مستقل مباشرة عبر متصفح لمعاينة الـ HTML الفعلي للسعر ومعدل التوظيف (بيئة العمل هنا لا تسمح بالوصول لموقع mostaql.com مباشرة). لذلك بنيت الاستخراج بطريقتين:

- **قائمة المشاريع** (`getProjectList`): تعتمد على الأصناف (classes) الموجودة في كودك الأصلي (`project-row`, `project__brief`, `project__meta`)، وهذه على الأرجح صحيحة لأنك كنت تستخدمها بالفعل.
- **صفحة المشروع الفردية** (`getProjectDetails`): اعتمدت على **البحث النصي** (regex) عن كلمتي "الميزانية" و"معدل التوظيف" بدل الاعتماد على classes معينة، لأن هذا أكثر ثباتًا مع تغييرات التصميم، لكنه **يحتاج تأكيدًا منك** على مشروع حقيقي قبل الاعتماد عليه بشكل كامل.

### كيف تتأكد أو تُصلح الاستخراج إن لم يعمل بشكل صحيح:

```bash
node src/debug.js "https://mostaql.com/project/xxxxx-عنوان-المشروع"
```

هذا يحفظ الصفحة كاملة في `data/debug.html`. افتحه وابحث (Ctrl+F) عن:
- `الميزانية` → لترى الشكل الفعلي لعرض السعر
- `معدل التوظيف` → لترى الشكل الفعلي لعرض تقييم العميل

إن كان الشكل مختلفًا عمّا تتوقعه الدوال، عدّل `parseBudgetFromText` و `parseEmploymentRate` في `src/scraper.js` (كلاهما موثّق بالتعليقات).

## تشغيلها مجانًا 24 ساعة بدون ترك جهازك مفتوحًا (GitHub Actions)

بدل ما تسيب `npm start` شغالة على جهازك، ممكن تخلي GitHub يشغّلها نيابة عنك مجانًا كل 5 دقائق، وده عن طريق ملف `.github/workflows/scrape.yml` الموجود جاهز داخل المشروع.

### الخطوات:

1. ارفع الكود على GitHub (اعمل ريبو جديد، ويفضل يكون **Public** لأن الـ Public repos عندها دقائق تشغيل مجانية غير محدودة على GitHub Actions، بينما الـ Private فيها حد شهري محدود قد لا يكفي مع تشغيل كل 5 دقائق):

    ```bash
    git init
    git add .
    git commit -m "أول نسخة من أداة مراقبة مستقل"
    git branch -M main
    git remote add origin https://github.com/USERNAME/mostaql-monitor.git
    git push -u origin main
    ```

    > لا تقلق بخصوص خصوصية توكن تيليجرام — ملف `.env` مُستثنى تلقائيًا عبر `.gitignore` ولن يُرفع أبدًا، والتوكن سيُخزَّن كـ "Secret" داخل GitHub بدل ملف `.env`.

2. من صفحة الريبو على GitHub: **Settings → Secrets and variables → Actions → New repository secret**، وأضف اثنين:
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`

3. اذهب لتبويب **Actions** في الريبو، ستجد وركفلو باسم "Mostaql Monitor" — فعّله إذا طلب منك ذلك (زر Enable workflow).

4. جرّبه فورًا يدويًا بدل انتظار أول 5 دقائق: من نفس تبويب Actions، اختر الوركفلو، ثم **Run workflow**.

5. من الآن فصاعدًا سيعمل تلقائيًا كل 5 دقائق، ويحفظ حالة `data/seen.json` و`data/matches.json` كـ commits جديدة في الريبو نفسه — تقدر تراجعها فيه أي وقت.

### ملاحظات مهمة:

- GitHub لا يضمن دقة التوقيت 100%، فقد يتأخر التشغيل بضع دقائق وقت الازدحام — هذا طبيعي ولا داعي للقلق.
- إذا مر الريبو بدون أي نشاط (commits) لمدة 60 يومًا، يوقف GitHub التشغيل التلقائي المجدول تلقائيًا — لكن بما أن الأداة نفسها تعمل commit لحالة الملفات كل تشغيلة، هذا لن يحدث طالما الأداة شغالة.
- إن فضّلت أن يبقى الريبو **Private**: هيشتغل برضو، بس عندك حد شهري لدقائق التشغيل المجانية (حوالي 2000 دقيقة/شهر)، وممكن يخلص قبل آخر الشهر مع التشغيل كل 5 دقائق. لو حصل كده، إما تخليه Public، أو تقلل التكرار (مثلاً كل 10 دقائق بدل 5) بتعديل `cron: "*/5 * * * *"` في ملف الوركفلو.



## هيكل الملفات

```
.github/workflows/
  scrape.yml              # تشغيل تلقائي مجاني كل 5 دقائق عبر GitHub Actions
.gitignore                # يستثني node_modules و .env من الرفع لـ GitHub
src/
  config.js              # كل الإعدادات القابلة للتعديل + نص خبرتك
  scraper.js              # جلب القائمة + جلب واستخراج تفاصيل كل مشروع
  filterAndProposal.js    # منطق الفلترة + توليد نص العرض
  storage.js              # حفظ المشاريع المعالجة والنتائج المطابقة (ملفات JSON)
  notifier.js             # إرسال إشعارات تيليجرام للمشاريع المطابقة
  index.js                # نقطة التشغيل الرئيسية (once أو cron)
  debug.js                # أداة مساعدة لمعاينة الـ HTML الخام لأي رابط
data/
  seen.json               # يُنشأ تلقائيًا: أرقام المشاريع التي تمت معالجتها
  matches.json            # يُنشأ تلقائيًا: المشاريع المطابقة + مسودة العرض
```

## تعديل نص خبرتك المستخدم في العرض

عدّل المتغير `MY_EXPERIENCE` في `src/config.js`.
